# Elyvo Assist — Руководство пользователя

> 🇬🇧 This guide is also available in [English](USER_GUIDE.md).

Elyvo Assist — это десктопный ИИ-ассистент для встреч, исследований и мозговых штурмов. Он работает как полупрозрачный оверлей поверх любого окна и вызывается горячей клавишей. Он может слушать ваш микрофон и системный звук, расшифровывать речь в реальном времени, видеть ваш экран и отвечать на вопросы с учётом контекста — оставаясь при этом скрытым от демонстрации и записи экрана.

Это руководство охватывает установку и обзор основных возможностей.

## Установка

Установщики и бинарные файлы публикуются в [GitHub Releases](https://github.com/pdasilem/elyvo-assist/releases). Скачайте файл, соответствующий вашей платформе, из последнего релиза. Все сборки 64-битные (`x86_64` / Apple Silicon).

Каждый релиз содержит для версии `X.Y.Z`:

| Платформа | Файл |
|-----------|------|
| Windows (рекомендуется) | `elyvo-assist-X.Y.Z-windows-x64-setup.exe` |
| Windows (MSI) | `elyvo-assist-X.Y.Z-windows-x64.msi` |
| macOS (Intel) | `elyvo-assist-X.Y.Z-macos-x64.dmg` |
| macOS (Apple Silicon) | `elyvo-assist-X.Y.Z-macos-arm64.dmg` |
| Debian / Ubuntu | `elyvo-assist-X.Y.Z-linux-x86_64.deb` |
| Arch / Manjaro | `elyvo-assist-X.Y.Z-1-x86_64.pkg.tar.zst` (+ `install.sh`) |

### Windows

1. Скачайте установщик `...-setup.exe` (NSIS) — или `.msi`, если в вашей организации предпочитают развёртывание через MSI.
2. Запустите его и следуйте инструкциям. Приложение устанавливается в `Program Files\Elyvo Assist`.
3. Запустите **Elyvo Assist** из меню «Пуск».

### macOS

1. Скачайте `.dmg` для вашего процессора — `macos-x64` для Intel, `macos-arm64` для Apple Silicon (M1/M2/M3 и новее).
2. Откройте образ диска и перетащите **Elyvo Assist** в папку **Applications**.
3. При первом запуске macOS может предупредить, что приложение от неизвестного разработчика. Кликните по приложению правой кнопкой → **Открыть** → **Открыть**, чтобы разрешить запуск.

> **Требования для Linux.** Elyvo Assist рассчитан на рабочий стол **KDE Plasma** на **Wayland**. Защита оверлея от захвата экрана реализована через KWin (композитор KDE), поэтому скрытие от демонстрации экрана работает только в KDE/KWin. Другие окружения (GNOME и т. д.) могут запускать приложение, но гарантии защиты от захвата на них не действуют. Также для захвата микрофона и системного звука необходим запущенный сеанс **PipeWire**.

### Linux — Debian / Ubuntu

```bash
sudo apt install ./elyvo-assist-X.Y.Z-linux-x86_64.deb
```

`apt` сам подтянет зависимости времени выполнения (WebKitGTK 4.1, OpenSSL 3, PipeWire). В старых версиях `apt` используйте `sudo dpkg -i ...`, а затем `sudo apt -f install`, чтобы доустановить недостающие зависимости.

### Linux — Arch / Manjaro

Самый быстрый путь — опубликованный скрипт установки, который скачивает пакет, устанавливает необходимые системные библиотеки и запускает `pacman` за вас:

```bash
curl -fsSL https://github.com/pdasilem/elyvo-assist/releases/latest/download/install.sh -o install.sh
bash install.sh
```

Скрипт поддерживает только системы на базе `pacman` и установит все недостающие пакеты времени выполнения (GTK3, WebKit2GTK 4.1, PipeWire, libayatana-appindicator и т. д.).

Предпочитаете сделать вручную? Скачайте `.pkg.tar.zst` и установите его напрямую:

```bash
sudo pacman -U elyvo-assist-X.Y.Z-1-x86_64.pkg.tar.zst
```

## Первый запуск

1. **Вход.** Elyvo Assist выполняет аутентификацию через вашу учётную запись Elyvo. Войти можно по **электронной почте и паролю** или через доступного **OAuth2-провайдера**. Новые пользователи могут создать аккаунт на том же экране.
2. **Онбординг.** Короткий мастер задаёт несколько вопросов о вас и о том, чем вы занимаетесь, чтобы ассистент мог персонализировать ответы. По желанию можно прикрепить любой файл с описанием вас; он используется только для того, чтобы дать ассистенту больше контекста о вас. Эти ответы можно позже пересмотреть и изменить в профиле.
3. **Заставка → оверлей.** После входа приложение сворачивается в небольшую заставку и работает в фоне. Вызвать оверлей чата можно в любой момент горячей клавишей переключения (по умолчанию `Ctrl+\`).

## Разрешения

Чтобы слушать и видеть ваш экран, Elyvo Assist нужны разрешения на уровне ОС. Откройте **Настройки → Разрешения**, чтобы просмотреть и выдать их. Приложение запрашивает:

- **Микрофон** — чтобы захватывать то, что вы говорите.
- **Системный звук** — чтобы захватывать собеседника в звонке (звук, выходящий из динамиков).
- **Запись / захват экрана** — чтобы функция *«Спросить про экран»* видела активное окно.
- **Универсальный доступ (Accessibility)** (macOS) — нужен для глобальных горячих клавиш и позиционирования оверлея.

Для каждого разрешения отображается статус — **Выдано**, **Отклонено** или **Не определено** — с кнопкой запроса. Если вы случайно отклонили разрешение, выдайте его в настройках ОС и снова откройте панель.

> В Linux захват системного звука использует PipeWire и desktop portal. Убедитесь, что PipeWire запущен (он включён по умолчанию в актуальных Manjaro и Ubuntu).

## Оверлей и горячие клавиши

Elyvo Assist управляется почти полностью с клавиатуры, чтобы вы могли пользоваться им, не отрываясь от встречи. Оверлей чата располагается поверх других окон, его можно перетаскивать, и он **скрыт от демонстрации и записи экрана** (см. [защиту окна](../README.md#window-protection-from-screen-sharing)).

Горячие клавиши по умолчанию (все переназначаются в **Настройки → Горячие клавиши**):

| Действие | По умолчанию | Что делает |
|----------|--------------|------------|
| Переключить видимость | `Ctrl+\` | Показать / скрыть оверлей Elyvo |
| Спросить Elyvo | `Ctrl+Enter` | Спросить про экран или текущий звук |
| Очистить чат | `Ctrl+R` | Очистить текущий разговор |
| Начать / остановить сессию | `Ctrl+Shift+\` | Начать или завершить сессию прослушивания |
| Переместить оверлей | `Ctrl+↑ / ↓ / ← / →` | Сдвинуть окно на экране |
| Прокрутить ответ | `Ctrl+Shift+↑ / ↓` | Прокрутить ответ вверх / вниз |

Чтобы переназначить, откройте **Настройки → Горячие клавиши**, кликните по сочетанию и нажмите новую комбинацию.

## Обзор возможностей

### Сессии

**Сессия** — это период, когда Elyvo активно слушает и удерживает контекст. Начать или остановить сессию можно сочетанием `Ctrl+Shift+\`. Во время сессии Elyvo захватывает ваш микрофон и (по желанию) системный звук, расшифровывает речь в реальном времени и хранит текущую расшифровку как контекст для ваших вопросов. Выбрать устройство ввода звука, а также протестировать уровни микрофона и системного звука можно в настройках.

### Спросить про экран или звук

Нажмите **Спросить Elyvo** (`Ctrl+Enter`), и Elyvo ответит, опираясь на то, что сейчас на экране, и недавний звук/расшифровку как контекст — удобно для «суммируй, что только что сказали», «что это за ошибка» или «набросай ответ на это». Также в любой момент можно просто ввести сообщение в поле чата.

### Режимы ИИ

**Режимы** позволяют настроить поведение ассистента под разные ситуации. У каждого режима есть собственный системный промпт и необязательный шаблон заметок. Управление — в разделе **Режимы**:

- Начните с **Галереи шаблонов** или создайте режим с нуля.
- Отредактируйте системный промпт, задав тон, роль и правила (например, режим «Продажный звонок» против «Стендап»).
- Прикрепите **файлы режима** — справочные материалы, которые ассистент должен учитывать в этом режиме.
- Отметьте один режим активным; всегда доступен общий режим по умолчанию.

### Фоновый ИИ-чат (Ambient)

Фоновый чат — это лёгкий, всегда доступный чат, который следует за вами по приложению и может быть привязан к проекту. Он входит в платный тариф (см. **Настройки → Оплата**).

### Проекты

**Проекты** группируют связанные сессии и дают ассистенту общий, постоянный контекст. Внутри проекта можно управлять:

- **Участники** — посмотреть, кто в проекте, и пригласить других по электронной почте (каждый приглашённый отображается как *ожидает*, пока не примет приглашение).
- **Память** — факты и контекст, которые ассистент должен помнить между сессиями этого проекта.
- **Правила** — указания, которым ассистент следует в этом проекте.

Когда *вас* приглашают в чужой проект, приглашение появляется вверху раздела **Проекты** с кнопками **Принять** / **Отклонить**. Фоновый чат можно привязать к проекту, чтобы ответы опирались на память и правила этого проекта.

### Дашборд и история

**Дашборд** — ваша домашняя страница: он показывает прошлые сессии в виде карточек и позволяет открыть детали сессии — её расшифровку, заметки и сводку встречи. Используйте его, чтобы просматривать встречи или возвращаться к ним.

### Память и самообучение

Elyvo становится лучше с использованием. В вашем **Профиле** можно просматривать и редактировать:

- **Память пользователя** — долгоживущие факты о вас и ваших предпочтениях, которые ассистент применяет везде.
- **Уточнения (Disambiguations)** — пояснения, которые ассистент усвоил (например, какой именно «Иван» или какой проект вы имеете в виду), чтобы он перестал ошибаться в догадках.

### Защита окна от демонстрации экрана

Оверлей намеренно невидим для захвата, чтобы вы могли пользоваться им во время совместного звонка, и он не попадал в трансляцию. Покрытие отличается по платформам — авторитетная таблица приведена в [основном README](../README.md#window-protection-from-screen-sharing). Кратко:

- **Windows** — скрыт от всех типов захвата «из коробки».
- **Linux (KDE / KWin)** — скрыт от *записи и демонстрации* экрана «из коробки»; чтобы скрыть его от статических *скриншотов* (Spectacle/PrintScreen), нужен одноразовый патч KWin, который нужно повторно применять после обновлений KWin.
- **macOS** — использует тот же нативный механизм защиты контента. Надёжно работает на **macOS 14 и более ранних**; на **macOS 15 и новее** скрытность **не гарантируется**, и оверлей может попасть в захват.

## Настройки

Откройте настройки из меню пользователя. Вкладки:

- **Общие** — основные параметры.
- **Горячие клавиши** — просмотр и переназначение всех сочетаний.
- **Профиль** — ваши ответы из онбординга, память пользователя и уточнения.
- **Безопасность** — параметры безопасности аккаунта.
- **Язык** — язык интерфейса / ответов.
- **Оплата** — ваша подписка и тариф (открывает платные функции, например фоновый ИИ-чат).

## Обновление

Скачайте свежий установщик для вашей платформы из [Releases](https://github.com/pdasilem/elyvo-assist/releases) и запустите его поверх текущей установки — настройки и вход сохранятся.

- **Arch / Manjaro:** заново запустите `install.sh` из последнего релиза или выполните `sudo pacman -U` для нового `.pkg.tar.zst`.
- **Debian / Ubuntu:** `sudo apt install ./elyvo-assist-<новая-версия>-linux-x86_64.deb`.
- **Windows / macOS:** запустите новый установщик / откройте новый DMG.

> Пользователям Linux KDE: повторно примените патч KWin для скриншотов после системного обновления KWin, если вы полагаетесь на защиту от скриншотов.

## Удаление

- **Windows** — *Параметры → Приложения → Установленные приложения → Elyvo Assist → Удалить*.
- **macOS** — перетащите **Elyvo Assist** из папки *Applications* в Корзину.
- **Debian / Ubuntu** — `sudo apt remove elyvo-assist`.
- **Arch / Manjaro** — `sudo pacman -R elyvo-assist`.

## Устранение неполадок

**Оверлей не появляется.** Убедитесь, что приложение запущено (проверьте трей / строку меню) и нажмите горячую клавишу переключения (`Ctrl+\`). На macOS проверьте, что выдано разрешение Accessibility, иначе глобальные горячие клавиши не сработают.

**Звук не захватывается.** Проверьте в **Настройки → Разрешения** доступ к микрофону и системному звуку, затем выберите нужное устройство ввода и воспользуйтесь тестом микрофона/системного звука в настройках. В Linux убедитесь, что PipeWire запущен.

**Оверлей всё ещё виден на скриншотах в Linux.** *Запись/демонстрация* экрана скрыты по умолчанию; для статических скриншотов нужен одноразовый патч KWin, описанный в [README](../README.md#window-protection-from-screen-sharing). Применяйте его повторно после обновлений KWin.

**Проблемы со входом.** Попробуйте альтернативный способ (почта/пароль или Google) и убедитесь, что системные часы выставлены верно — OAuth и проверка токенов чувствительны ко времени.

По остальным вопросам создайте issue в [репозитории релизов](https://github.com/pdasilem/elyvo-assist/issues).
